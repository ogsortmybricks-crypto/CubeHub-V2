# Features

This document defines the functional capabilities of CubeHub. Every feature should support one or more of the project's core pillars: Learning, Practice, Community, Competition, History, Research, AI, or Developers.

---

# Learning

CubeHub should be the best place on the internet to learn speedcubing.

## Algorithm Database

Every algorithm should have:

* Interactive 3D viewer
* Multiple execution variants
* Fingertricks
* Move counts (HTM, STM, QTM)
* Estimated TPS
* Recognition guide
* Historical information
* Community ratings
* AI explanations
* Alternative algorithms

## Method Explorer

Support all major solving methods.

Each method includes:

* History
* Inventor
* Learning roadmap
* Pros and cons
* Example solves
* Community discussions
* Related methods

## Recognition Trainer

Generate randomized practice cases.

Features:

* Random AUFs
* Random cube rotations
* Adjustable lighting
* Recognition timer
* Performance statistics
