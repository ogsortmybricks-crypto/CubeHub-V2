# Product Roadmap

CubeHub should grow in carefully planned stages.

Each version should solve one major problem exceptionally well before expanding.

---

# Version 1

## Learn

Objective:

Become the best algorithm learning platform.

Features:

* Algorithm database
* 3D cube viewer
* Method explorer
* User accounts
* Personal collections
* Basic practice
* Search

Success Metric:

Users prefer CubeHub over static algorithm sheets.

---

# Version 2

## Improve

Objective:

Become the best practice platform.

Features:

* AI practice planner
* Statistics
* Recognition trainer
* Daily practice
* Goals
* Progress tracking

Success Metric:

Daily active users.

---

# Version 3

## Collaborate

Objective:

Become the place where knowledge is published.

Features:

* Repositories
* Forks
* Pull requests
* Discussions
* Wiki
* Version history

Success Metric:

Creator adoption.

---

# Version 4

## Community

Objective:

Become the social hub of speedcubing.

Features:

* Clubs
* Journals
* Solve stories
* Activity feed
* Events
* Messaging

Success Metric:

Community engagement.

---

# Version 5

## Ecosystem

Objective:

Become infrastructure for speedcubing.

Features:

* Public API
* SDK
* Smart cube integration
* Competition integrations
* Mobile app
* Developer tools

Success Metric:

Third-party integrations.

---

# Version 6+

Continue expanding into AI, research, education, and emerging technologies while preserving the core mission of the platform.
